■ 在庫管理の概念区分（ドキュメントより確認済み）

論理在庫（Logical Inventory）
EFS/Core が SoT。
管理単位:「場所ID × 場所属性ID」の論理ロケーション層
例: 物流_PK-BtoC (101)、調整 (191/291/391)
※棚番・ケース単位の管理は行わない（原本明記）
L1/L2/L3 引当階層・TRS（積送在庫）を含む
物理在庫（Physical Inventory）
WMS が SoT。
管理単位: 棚番（実ロケーション）レベル
入荷検品・出荷検品・棚卸・差異報告の起点
B在庫（ブランド倉庫）/ C在庫（EC倉庫）を保持
EFS は「場所ID × 場所属性ID」でこれを論理化

05推奨方針
推奨 Option A — Core = 論理在庫、WMS = 物理在庫 + 棚卸

根拠①: AS-IS ドキュメントで「EFS は棚の管理を行わない・ケースの管理を行わない」と明記されており、現行設計が既に Option A の構造で稼働しています。

根拠②: open-decisions の最大の懸念事項「WMS 側の棚卸機能を確認」は解決済みです。WMS は棚卸ケース1〜3をバッチ（STELCD11/17）含め実装・稼働しており、廃棄処理確定を EFS が担う合理的な分業が成立しています。

根拠③: Option B は EFS に棚番管理データモデルを新設する必要があり、現行 DDL に対応カラムが存在しないため、EFSO プロジェクトのスコープ拡大リスクが高い。
