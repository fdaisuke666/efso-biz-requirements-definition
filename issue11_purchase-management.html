重要: EFS は「発注〜月次支払額確定」まで管理。出金管理（厳密な買掛管理）は EFS スコープ外で会計システムが担当。GWCS は「外注入荷報告（仕入）」をまだ保持しており、廃止時の Core 側への移行可否が未確認。
02論点の整理
論点 A
機能の重複
EFS と GWCS の仕入関連機能が一部重複。GWCS 廃止時にどちらに集約するかを先に決めないと仕入機能が宙に浮く。
論点 B
EDI 連携（IL-19）
Option A（Core 化）の場合、取引先 EDI（IL-19）が必要。Core が発注ワークフローを持つなら IL 経由で EDI 接続を設計する必要がある。
論点 C
#23 ドメイン分離
#23（売上/仕入の独立ドメイン化）と連動。finance を独立ドメインにするかどうかの方針と整合が必要。

観点Option A — Core に含めるOption B — GWCS に残す

中長期保守性◎ GWCS 廃止後も Core が自己完結。保守コスト低✕ GWCS を維持し続ける保守コスト・技術負債が蓄積
GWCS 廃止リスク◎ GWCS 廃止のブロッカーを除去できる✕ 仕入機能を理由に GWCS 廃止が遅延するリスク
#23 との整合◎ finance 独立ドメイン化と親和性高い△ finance を独立ドメイン化しても仕入は GWCS に分散
05推奨方針
推奨Option A — EFSO Core に仕入管理を含める

根拠①（GWCS 廃止との整合）: GWCS は段階的廃止フェーズにあります。仕入管理を GWCS に残すと廃止のブロッカーになり、中長期で技術負債が増大します。

根拠②（現行設計の自然な延長）: EFS は既に発注管理・月次締め・支払管理の主要機能を保持しており、Core 化は現行設計の自然な延長です。

根拠③（#23 との連動）: finance を独立ドメインとして分離する方向で #23 の方針 Fix と同時に判断することで、整合した設計が可能になります。

07決定を阻む不確定要素（ブロッカー）

以下の5項目が未解決のまま残ると、Option A を採用しても詳細設計・移行計画の策定に着手できません。優先度順に解消してください。

B-1 GWCS 保持の仕入機能の全量が未調査 CRITICAL
「外注入荷報告（仕入）」以外にも GWCS 側に残っている仕入機能があるかが未確認。EFS 側との重複範囲が特定できていないため、Core へ移行すべき機能の全量が不明。
移行スコープが確定できず、工数見積もりができない。機能の取りこぼしが発生するリスクがある。
GWCS の仕入関連機能を棚卸しするための調査を2週間以内に実施。EFS との重複・非重複を一覧化する。
Owner: CTO + 開発チーム / 期間: 2週間以内
B-2 IL-19（取引先 EDI）の実現性が未確認 CRITICAL
Option A 採用時に必須となる IL-19 の接続先・プロトコル・実装工数が未確認。EDI 接続が技術的・コスト的に実現可能かが不明。
IL-19 が重い場合、Option A のコスト試算が大きく変わり、選択肢の再評価が必要になる可能性がある。
EDI 接続先（取引先側）の要件を確認し、IL-19 の技術的実現性を POC レベルで評価する。
Owner: CTO + 開発チーム
B-3 #23（financeドメイン分離）の方針が未決定 HIGH
finance を独立ドメインとして分離するか、受注ドメインに統合したまま保持するかの方針が #23 で未決定。#11 と #23 を別々に判断するとアーキテクチャが不整合になるリスクがある。
#11 だけ先に決めても #23 と矛盾する設計になる可能性がある。
#11 と #23 を同一の意思決定セッションで確定する。CTO がオーナーとなりアーキテクチャ判断会を設定。
Owner: CTO + dev
B-4 消化仕入商品のフロー設計が未確認 HIGH
消化仕入商品は「発注なし・出荷時に売上と仕入が同時計上」という特殊フロー。Core に移行した場合にこのロジックが正しく再現できるか設計上未確認。
消化仕入比率が高い場合、移行コストが跳ね上がる。フロー設計漏れにより本番障害に直結するリスクがある。
消化仕入の同時計上ロジックを EFSO Core に移植可能か、開発チームが設計レビューを実施する。
Owner: CTO + 開発チーム
B-5 アットトバス連携の継続可否が未確認 MEDIUM
支払報告書の郵送をアットトバスが担っているが、IL 経由への移行要否・継続可否がビジネス側と合意できていない。外部ベンダー依存の部分であり Core のスコープ設計に影響する。
TO-BE でアットトバス連携を IL に含める場合と含めない場合で IL 設計の規模が変わる。
調達・ビジネスサイドでアットトバス連携の継続方針を確認。IL-19 の設計と合わせて判断。
Owner: ビジネス・調達 / 連携: CTO
08TO-BE 推奨アプローチ & ネクストアクション
