結論: EFS-GWCS 間の仕入機能は重複ではなく役割分担。EFS が発注・マスタ・計上・支払・月締・報告書の全てを保有しており、GWCS は 3 本の仕入実績 IF を提供するのみ。したがって「新 Core は EFS 側の仕入管理機能をそのまま引き継ぐ」のが自然な設計となる。
02論点の整理（実装方針判断レベル）
前提の変化: GWCS 存続が確定し、周辺システム（GWCS / WMS / アットトバス / 管理会計）の改修なしを前提としたため、以前想定していた「機能の重複」「EDI 新設（IL-19）」等の実装方針ブロッカーは実質消滅した。残るのは設計フェーズで解消すべき課題のみ。
論点 A — 解消済
機能の重複
AS-IS 再調査により、EFS-GWCS 間は重複ではなく役割分担と確定。EFS が主体、GWCS は 3 本の仕入実績 IF を提供するのみ。新 Core は EFS 側をそのまま引き継げばよい。
論点 B — 解消済
EDI 連携（IL-19）
周辺システム改修なしの前提により、取引先 EDI（IL-19）の新規構築は今回スコープ外。既存のアットトバス郵送フローを継続利用するため、EDI 未着工でも実装方針は成立する。
論点 C — 残存
#23 ドメイン分離
#23（finance 独立ドメイン化）は残存。ただし仕入管理を EFS から引き継ぐ方針自体には影響せず、Core 内部のドメイン配置問題に閉じる（設計フェーズ判断）。

05推奨方針
確定可Option A — 現行 EFS の仕入管理機能を新 Core にそのまま引き継ぐ

根拠①（現行 EFS が主体で成立済み）: as-is/finance 調査により、EFS が発注・マスタ・計上・支払・月締・報告書の全てを既に保有していることが確認された。GWCS は 3 本の仕入実績 IF を提供するのみで、両者は「重複」ではなく「役割分担」の関係にある。

根拠②（周辺システム改修なしの前提で成立）: GWCS 存続確定・WMS / アットトバスの改修なしという前提のもとでも、EFS→Core 引継モデルは現行 IF 契約をそのまま維持できる。IL-19（取引先 EDI）の新設も今回スコープ外で成立する。

根拠③（実装方針判断ブロッカーは消滅）: 従来想定していた「機能重複の集約」「EDI 新設」「GWCS 廃止順序」等の判断ブロッカーは前提変化により解消。残るのは設計フェーズで順次解消する課題のみで、方針 Fix を妨げない。

09補足 — 用語解説

本資料に登場する専門用語を「たとえ」付きで解説します。

仕入管理
商品を仕入先から買う（または預かる）一連の業務。発注・入荷・検収・支払・請求までを含む。
たとえ: レストランが食材を業者から仕入れるのと同じ流れ。「何を何個発注→届いたか確認→今月いくら払うか計算→支払」の一連の業務。EC では売上を支える根幹業務。
買取仕入 / 消化仕入
買取仕入 = 先に買い取って在庫にする通常の仕入。
消化仕入 = 商品を預かり、売れた分だけ仕入計上する仕入形態。売れ残りはメーカーに返品。
たとえ:
・買取仕入 = スーパーで野菜を 100 個買って店頭に並べる（売れなくても自腹）
・消化仕入 = 委託販売の雑貨屋。お店にあるが売れた分だけ作家さんにお金を払う。アパレルでは消化仕入の比率が高い。
EDI（電子データ交換）
取引先との間で発注・納品・請求などのデータを電子的にやり取りする仕組み。紙・FAX・メール添付ではなく、システム間で直接連携する。
たとえ: FAX で発注書を送っていたのを、システム同士で自動的にデータを送受信する仕組みに変えたもの。人手が要らず、速く正確。取引先ごとにフォーマットが違うのが難点。
IL-19（取引先 EDI コネクタ）
新システムで想定されている取引先 EDI 用の統合レイヤー（IL）機能の識別番号。取引先ごとのフォーマット差異を吸収する役割。
たとえ: 各取引先が別々の言語を話す状態を、通訳（IL-19）を間に置いて Core が 1 つの共通言語で話せばよいようにする仕組み。
GWCS（店舗/倉庫基幹システム）
店舗・倉庫の業務を担う現行の基幹系システム。POS 売上集約、店舗棚卸、配分指示、在庫予算、管理会計など広範な機能を持つ。存続が確定している。
たとえ: 店舗・倉庫側の「司令塔」。EC 側の EFS（→ Core）と役割分担していて、今回は仕入機能をどちらに寄せるかが論点。
外注入荷報告（仕入）
外部倉庫や仕入先メーカーから「入荷が完了しました」という報告を受け取り、仕入として計上する処理。現行は GWCS 側で実施。
たとえ: 外部倉庫の担当者が「荷物を受け取りました」という報告書を本社に送るイメージ。この報告をトリガーにして仕入計上が走る。
アットトバス（支払報告書郵送サービス）
仕入先への「今月はこれだけお支払いします」という支払報告書を印刷・郵送代行する外部サービス。EFS がデータを連携して郵送を委託している。
たとえ: 自社で印刷・封入・切手貼り・投函をやる代わりに、外部業者に一括委託している状態。電子メール送付やポータル掲載に切り替えるかが論点（B-5）。
finance ドメイン / 売上仕入
売上計上・仕入計上・請求・支払などお金の流れを扱う業務領域のまとまり。#23 で独立ドメインにするかが議論中。
たとえ: 会社の「経理部門」に相当する機能群。現在は受注ドメインの中に混在しているが、独立した部門（ドメイン）として切り出すかを検討中。#11 の配置先が変わる。
